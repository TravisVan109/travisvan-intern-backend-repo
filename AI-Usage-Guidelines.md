# AI Usage Guidelines

## Goal

The goal of this activity is to understand how to use AI tools responsibly and effectively while protecting sensitive information and maintaining critical thinking. AI tools can improve productivity, but they must be used carefully to avoid mistakes or data privacy issues.

# Research

## AI Tools Used in My Role

As a backend intern, AI tools can help support tasks such as:

- Generating example code or debugging suggestions
- Explaining technical concepts or documentation
- Improving writing for documentation or communication
- Brainstorming ideas for features or improvements

Examples of AI tools include ChatGPT, code assistants, and AI-powered developer tools that help with coding and research.

## Benefits and Risks of Using AI

### Benefits

- Helps generate ideas and solutions quickly
- Improves productivity when researching or learning new concepts
- Assists with writing, documentation, or code suggestions
- Helps explain complex technical topics

### Risks

- AI-generated information may be incorrect or outdated
- Over-reliance on AI can reduce critical thinking
- Sensitive or confidential data could be exposed if entered into AI tools
- AI responses may not fully understand the context of a project

Because of these risks, AI should be used as a **support tool**, not a replacement for human judgement.

## Information That Should Not Be Entered into AI Tools

To protect company and user privacy, the following information should never be entered into AI tools:

- Confidential company information
- User data or personal information
- Internal systems, credentials, or API keys
- Private business strategies or internal documents
- Any sensitive project details not meant for public sharing

Protecting sensitive data is essential to maintain security and trust.

## Validating AI-Generated Content

AI-generated outputs should always be checked carefully before being used. Some ways to validate AI responses include:

- Reviewing the information critically
- Comparing results with official documentation
- Testing generated code before using it
- Asking teammates or supervisors when unsure

Fact-checking ensures that AI-generated content is accurate and reliable.

# Reflection

## When to Use AI vs Personal Skills

AI can be useful when:

- Researching unfamiliar technical topics
- Brainstorming ideas or possible solutions
- Improving written communication
- Getting explanations for complex concepts

However, it is important to rely on personal skills when:

- Making final decisions
- Implementing important code changes
- Handling sensitive or confidential information
- Evaluating whether AI-generated information is correct

AI should support learning and productivity rather than replace critical thinking.

## Avoiding Over-Reliance on AI

To avoid relying too heavily on AI, I will:

- Use AI mainly as a **starting point for ideas or explanations**
- Verify information using official documentation
- Try solving problems independently before asking AI
- Review AI-generated results carefully before using them

This approach helps maintain strong problem-solving skills while still benefiting from AI tools.

## Ensuring Data Privacy

To protect data privacy when using AI tools, I will:

- Avoid entering any confidential company or user data
- Use AI tools only for general questions or learning purposes
- Remove or anonymize sensitive details when discussing technical problems
- Follow company policies regarding data protection

These steps help ensure that AI is used safely and responsibly.
