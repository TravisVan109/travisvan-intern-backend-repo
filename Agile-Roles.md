# Agile Team Roles

# Research

## Key Roles in an Agile Team

### **1. Developers**

Developers are responsible for:

- Writing and maintaining code
- Implementing features and fixing bugs
- Collaborating on technical design
- Ensuring the product is functional and reliable

They work closely with the PM, QA, and other team members throughout the workflow.

### **2. QA (Quality Assurance)**

QA engineers ensure the product meets quality standards by:

- Testing new features
- Identifying bugs or issues
- Creating test plans and automation (in some teams)
- Verifying fixes before release

Their role ensures the final product is stable and user-friendly.

### **3. Product Manager (PM)**

The PM is responsible for:

- Managing the product vision and priorities
- Creating and maintaining the product backlog
- Helping clarify requirements and acceptance criteria
- Making sure the product meets user needs

PMs decide what to build and why, while developers figure out how to build it.

### **4. Agile Project Manager (Agile PM) / Scrum Master**

The Agile PM supports the team by:

- Facilitating Agile ceremonies (stand-ups, planning, retrospectives)
- Removing obstacles that block progress
- Helping the team improve processes
- Encouraging collaboration and communication

They focus on team health and efficient workflow.

### **5. Supporting Roles**

Other roles involved in Agile teams include:

- **UX / UI Designers:** Create user journeys, layouts, and design assets.
- **Customer Support:** Shares user feedback and common issues.
- **Stakeholders:** Provide business goals and high-level direction.
- **DevOps / Infrastructure:** Maintain deployment pipelines and systems reliability.

## Collaboration in Kanban vs Scrum

### Kanban

- Emphasizes **continuous flow** rather than fixed sprints
- Roles collaborate as tasks move through the board
- Frequent updates but fewer formal ceremonies
- Team members pull work when ready, based on priority

### Scrum

- Works in **time-boxed sprints**
- Involves formal ceremonies (Planning, Stand-ups, Review, Retrospective)
- PM defines sprint goals; Scrum Master facilitates process
- Developers and QA deliver a set of features each sprint

Both methodologies rely on communication and collaboration, but Kanban is more flexible while Scrum is more structured.

My responsibilities overlap with several roles:

- **PM:** I clarify requirements or ask questions when tasks are unclear.
- **QA:** I work with QA to fix issues found during testing.
- **Developers:** I collaborate with other developers when solving technical problems or reviewing code.
- **UX:** Sometimes I adjust APIs or data flows to support UX requirements.

Good communication helps avoid confusion and ensures features are built correctly.
