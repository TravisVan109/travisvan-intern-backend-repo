# AI Tools – Reflection

## AI Tools I Tried

During this task, I used all three AI tools:

- **GitHub Copilot** – for code suggestions directly in VS Code
- **ChatGPT** – for planning, brainstorming ideas, and checking problems
- **Claude AI** – for extra coding assistance and explanations

I already use these tools regularly, so this task helped me reflect on how I use them in my workflow.

## What Worked Well

### GitHub Copilot

- Great for **quick code suggestions** as I type.
- Helps speed up writing boilerplate code.
- Useful for reminding me of syntax or common patterns.

### ChatGPT

- Best for **planning, brainstorming, and breaking down problems**.
- Very helpful when I need an explanation or want to understand a concept more deeply.
- Good for reviewing code or giving high-level guidance.

### Claude AI

- Strong at **clear explanations** and rewriting code more cleanly.
- Good when I want an alternative answer or need help understanding tricky logic.

## What Didn’t Work So Well

- AI suggestions can sometimes be **incorrect or incomplete**, so I still have to check everything carefully.
- Copilot sometimes predicts code that doesn’t match what I actually want.
- ChatGPT and Claude sometimes give answers that sound correct but aren’t practical for my exact codebase.
- AI cannot fully understand project context unless I explain it clearly.

## When AI Is Most Useful for Coding

In my experience, AI is most helpful when:

- I need a quick **code example** or starting point
- I want help **debugging** or checking why something isn’t working
- I’m learning a **new concept** and need it explained simply
- I want to **improve my progress** or reduce workload on repetitive tasks
- I need a clearer way to structure or review my code

AI works best as an **assistant**, not a replacement for thinking. I still need to understand the code myself.

## Summary

Using multiple AI tools helps me:

- Work faster
- Understand new concepts
- Review and check my work
- Reduce time spent on repetitive tasks

AI supports my workflow, but I make sure to think critically and verify the results before using them.
