# VS Code Productivity – Useful Shortcuts

Below are the top VS Code shortcuts I researched and tested.  
I grouped them into categories to make them easier to understand.

# Reflection

## My Top Shortcuts
My top 10 shortcuts that I find most useful are:
1. **F5** – Start debugging  
2. **Shift + F5** – Stop debugging  
3. **F9** – Toggle breakpoints  
4. **Shift + Alt + ↓ / ↑** – Duplicate lines (my most-used “copy-paste style” shortcut)  
5. **Ctrl + /** – Comment/uncomment code fast  
6. **Alt + Click** – Multiple cursors  
7. **Ctrl + B** – toggle sidebar
8. **Ctrl + Shift + K** – Delete line quickly  
9. **Ctrl + P** – Jump between files without using the mouse  
10. **Ctrl + Tab** – Cycle through open files  

These shortcuts help me move faster without relying on the mouse.


## Which Shortcuts Will Be Most Useful Daily?
The following shortcuts will help me the most in my daily work:
- **Shift + Alt + ↓ / ↑** → duplicating lines quickly  
- **Ctrl + /** → quick commenting/uncommenting  
- **Ctrl + `** → easy terminal access  


These are actions I do constantly while coding, so knowing the shortcuts saves a lot of time.


## How Keyboard Shortcuts Improve Productivity
Using keyboard shortcuts improves productivity because:
- I avoid switching between keyboard and mouse  
- Navigation becomes much faster  
- Editing code feels smoother and more efficient  
- I can perform repetitive tasks (like commenting or duplicating lines) instantly  
- It reduces mental overhead when working in large codebases  

Learning shortcuts helps me stay in a "flow state" longer, which improves both speed and focus.

## Summary
Learning these VS Code shortcuts will help me:
- Work faster  
- Avoid unnecessary mouse movements  
- Stay focused  
- Handle files, terminals, and debugging efficiently
