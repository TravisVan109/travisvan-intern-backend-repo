# Git Knowledge – Reflection

## My Experience With Git
I have used Git before.
My experience includes:
+ Using GitBash, GitLab from my previous company
+ Using GitHub for assignments and uni work
+ Basic commands like git add, git commit, git push, and git pull

Using Git as part of my daily workflow as a software engineering student and intern

## Git Client I Chose
I decided to use Git in IDE (E.g. VS Code), because:

+ It integrates directly into my development workflow

+ I can see changes, commits, and diffs visually

+ It makes staging files easier

+ It helps avoid mistakes by showing clear UI indicators

Sometimes I also use GitHub Desktop, GitBash or even Git command line... But using directly from IDE still prefer

## Most Interesting Thing I Learned Today
The most interesting thing I learned is how Git tracks changes locally before anything is pushed to a remote repository.
It reminded me that:

+ Git works offline

+ My work is always saved through commits

+ Nothing is synced to GitHub until I push
