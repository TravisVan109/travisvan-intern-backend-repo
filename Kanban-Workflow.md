# Kanban Workflow

# Research

## How a Kanban Board Works

A Kanban board is a visual system used to track work as it moves through different stages. It helps teams:
- See the status of all tasks at a glance  
- Identify bottlenecks  
- Keep workflow organized  
- Maintain focus on the most important tasks  

The board is divided into columns representing each stage of the workflow.


## Common Kanban Columns

Typical columns include:

- **Backlog / Not Started** – Tasks that are planned but not yet worked on  
- **In Progress** – Tasks currently being worked on  
- **Blocked** – Tasks that cannot move forward due to an issue  
- **Ready for Review** – Tasks completed by the developer and awaiting review  
- **Approved / Done** – Tasks completed and verified  

These columns help track progress step-by-step.

## How Tasks Move Through the Board

Tasks move across the board as work progresses:

1. Added to **Backlog / Not Started**
2. Moved to **In Progress** once work begins  
3. If a blocker appears, moved temporarily to **Blocked**  
4. When finished, moved to **Ready for Review**  
5. After approval, moved to **Done**  

Developers are usually responsible for updating the Kanban board so the whole team has accurate visibility.

## Benefits of Limiting Work in Progress (WIP)

Limiting WIP helps prevent overload and improves focus by:

- Encouraging developers to finish existing work before starting new tasks  
- Reducing task switching  
- Making bottlenecks easier to identify  
- Helping maintain a smooth workflow  

Fewer simultaneous tasks = better quality and faster completion.

# Reflection

## How Kanban Helps With Priorities and Avoiding Overload

Kanban helps prioritize work by visually showing:
- What tasks are active  
- What tasks are blocked  
- What tasks need attention  

Seeing the workflow clearly helps avoid starting too many tasks at once and keeps the team focused on finishing high-priority items before taking on new ones.

## Improving My Workflow Using Kanban Principles

I can improve my workflow by:

- Keeping tasks small and clearly defined  
- Updating the board frequently so progress is always accurate  
- Avoiding multitasking by focusing on one task in **In Progress** at a time  
- Using the **Blocked** column to highlight issues early instead of keeping them to myself  

These habits help maintain clarity and reduce delays.
